# Workflow Example: Managing a Campaign in a Project Management Tool

This document outlines a detailed workflow for using a project management tool (like Asana, Monday.com, or Wekan) to manage a marketing campaign. This demonstrates your ability to organize complex projects and coordinate with a team.

**Scenario:** The marketing team is launching a new "Spring Roofing Promotion" direct mail campaign in three weeks. You are responsible for building out the project plan in the company's project management tool to ensure all tasks are tracked and completed on time.

---

### Phase 1: Project Setup & Scaffolding

1.  **Log In and Choose a Workspace:** Access the project management tool and ensure you are in the correct team workspace (e.g., "Marketing").
2.  **Create the New Project:**
    *   Click the "Create Project" or `+` button.
    *   Select an option to create a new project from scratch. Choose a **Board layout** (Kanban-style) as it's visually intuitive for tracking stages.
    *   **Name the Project:** "Q2 Campaign: Spring Roofing Direct Mail".
    *   Set the project privacy to be visible to the Marketing & Sales team members.
3.  **Define the Stages (Columns/Lists):** A project moves through stages. Create columns for each distinct phase of the campaign.
    *   **Column 1: To Do / Planning:** All tasks start here.
    *   **Column 2: In Progress:** Tasks currently being worked on.
    *   **Column 3: In Review:** Tasks that are complete but need approval (e.g., design mockups).
    *   **Column 4: Done:** Completed tasks.
    *   **Column 5: Blocked / On Hold:** Tasks that cannot proceed for an external reason.

### Phase 2: Task Breakdown

This is the most critical part: breaking a large goal ("launch a campaign") into small, actionable tasks.

1.  **Brainstorm & Add Tasks:** Go through the campaign lifecycle and add each individual task as a "card" or "item" in the "To Do / Planning" column.
    *   `Finalize promotion details (discount %, duration)`
    *   `Define target mailing area and audience size`
    *   `Export mailing list from MarketSharp CRM`
    *   `Write copy for postcard`
    *   `Design postcard (Version A)`
    *   `Get design approval from Sales Manager`
    *   `Get printing quotes from 3 vendors`
    *   `Select printer and submit final design`
    *   `Postcards mailed by vendor`
    *   `Prepare sales team for incoming calls`
    *   `Track response rate`

### Phase 3: Adding Detail and Dependencies

1.  **Assign Owners:** Click into each task and assign it to the responsible team member (e.g., assign "Write copy for postcard" to the Marketing Manager).
2.  **Set Due Dates:** Work backward from the campaign launch date. The mailing needs to go out in two weeks, so the printing must be done by then. Set due dates for each task accordingly. A task like "Get design approval" might have a due date of this Friday.
3.  **Add Descriptions & Checklists:**
    *   For a task like `Export mailing list from MarketSharp CRM`, you can add a description: "Export homeowners in the 49002 zip code who have not been contacted in the last 12 months."
    *   You can also add a sub-task checklist: `[ ] Log in to MarketSharp`, `[ ] Apply filter`, `[ ] Export as CSV`, `[ ] Clean data`.
4.  **Set Dependencies (Crucial Feature):**
    *   Find the "Dependencies" feature.
    *   Mark `Design postcard` as **"blocked by"** `Write copy for postcard`.
    *   Mark `Get design approval` as **"blocked by"** `Design postcard (Version A)`.
    *   This ensures that team members are notified when a task they are waiting on is complete and that work happens in the correct order.

### Phase 4: Templating and Maintenance

1.  **Save as a Template:** Once the project structure is built out, find the option to "Save Project as Template." Name it "Direct Mail Campaign Template." The next time the team runs a similar campaign, you can start from this template in one click instead of rebuilding it from scratch. This demonstrates foresight and an efficiency mindset.
2.  **Monitor Progress:** Your ongoing role would be to monitor the project board daily. If you see a task that is past its due date, you would follow up with the assignee. If a task is moved to "Blocked," you would ask what the blocker is and help resolve it.

This structured approach shows that you can translate a business goal into a clear, actionable plan that a team can follow, which is a core competency of any operations or administrative role.
